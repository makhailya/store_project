# 🛒 Store Project

## 📦 Описание проекта
Этот проект — базовая версия системы для управления товарами и категориями.  
Он реализует классы `Product` и `Category`, которые позволяют описывать ассортимент магазина, управлять остатками, проводить валидацию данных и выполнять расчёты.

## 📈 История разработки (по урокам)

### 🧩 Lesson 2 — Классы
* Реализованы базовые классы `Product` и `Category`.
* Добавлен подсчёт общего количества категорий и товаров (`category_count`, `product_count`).

### ✨ Lesson 3 — Магические методы
* Реализован `__str__` для `Product` и `Category` для удобного вывода.
* Реализован `__add__` для `Product` для сложения общей стоимости товаров на складе.

### 🧬 Lesson 4 — Наследование и ограничения
* Реализованы классы-наследники `Smartphone` и `LawnGrass`.
* Доработан `__add__`: теперь выбрасывает `TypeError` при попытке сложить товары разных классов.
* Доработан `add_product()`: проверяет, что добавляемый объект является `Product` или его наследником.

### 🧱 Lesson 5 — Абстракции и миксины
* Создан абстрактный класс `BaseProduct` для задания общей структуры.
* Реализован миксин `CreationLoggerMixin` для логирования создания объектов.
* Класс `Product` наследует `BaseProduct` и `CreationLoggerMixin`.
* Добавлены приватные атрибуты (`__price`, `__products`).
* Реализованы геттеры, сеттеры (с валидацией цены) и `classmethod new_product()`.

### 🚀 Lesson 6 — Улучшение архитектуры, валидация и расчётные методы
* **Валидация:** Добавлена проверка `quantity > 0` при инициализации `Product` (выбрасывает `ValueError`).
* **Расчётная логика:** Реализован метод `middle_price()` в `Category`, который корректно обрабатывает пустые списки товаров (возвращает `0.0`).

## ✨ Ключевые возможности
* **Объектная модель:** Реализованы классы `Product` и `Category` для управления товарами.
* **Учёт:** Ведется автоматический подсчёт общего числа категорий (`category_count`) и уникальных товаров (`product_count`).
* **Наследование:** Архитектура расширена за счет дочерних классов `Smartphone` и `LawnGrass`.
* **Абстракция:** Используется абстрактный класс `BaseProduct` для единого интерфейса и миксин `CreationLoggerMixin` для логирования.
* **Перегрузка операторов:**
    * `__str__`: кастомное отображение объектов (`"iPhone, 120000.0 руб. Остаток: 5 шт."`).
    * `__add__`: сложение общей стоимости товаров на складе (e.g., `product_a + product_b`).
* **Валидация:**
    * **Цена:** Сеттер для `price` не позволяет установить нулевую или отрицательную цену.
    * **Количество:** Исключение `ValueError` при попытке создать товар с `quantity <= 0`.
    * **Типы:** Исключение `TypeError` при попытке сложить (`__add__`) товары разных классов.
* **Расчётные методы:**
    * `middle_price()`: Метод в `Category` для расчёта средней цены, корректно обрабатывающий пустые категории (возвращает `0.0`).

## 💡 Пример использования

```python
from src.category import Category
from src.product import Product
from src.smartphone import Smartphone

# Создание товаров
iphone = Smartphone("iPhone 15", "Смартфон Apple", 120000.0, 15, 2.8, "Pro", "512Gb", "Black")
samsung = Product("Galaxy S24", "Смартфон Samsung", 95000.0, 10)

# Создание категории
smartphones = Category("Смартфоны", "Мобильные устройства", [iphone, samsung])

# Вывод информации (магический метод __str__)
print(smartphones)
# "Смартфоны, количество продуктов: 25 шт."

print(iphone)
# "iPhone 15, 120000.0 руб. Остаток: 15 шт."

# Сложение общей стоимости (магический метод __add__)
total_cost = iphone + samsung
print(total_cost)
# 2750000.0 (120000*15 + 95000*10)

# Расчёт средней цены
print(smartphones.middle_price())
# 107500.0

    broken_product = Product("Брак", "Сломан", 100.0, 0)
except ValueError as e:
    print(e)
    # "Товар с нулевым количеством не может быть добавлен"
