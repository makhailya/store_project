# Store Project

## 📦 Описание проекта
Этот проект — базовая версия системы для управления товарами и категориями.  
Он реализует классы `Product` и `Category`, которые позволяют описывать ассортимент магазина и учитывать количество товаров и категорий.

---

## ⚙️ Реализованный функционал

### 🧱 Класс `Product`
Класс описывает отдельный товар и содержит следующие атрибуты:
- **name (str)** — название товара  
- **description (str)** — описание  
- **price (float)** — цена с возможностью указания копеек  
- **quantity (int)** — количество товара на складе  

Пример использования:
```python
from src.product import Product

iphone = Product("iPhone 15", "Смартфон Apple", 120000.0, 15)

### 🗂️ Класс Category

Класс описывает категорию товаров и хранит список объектов класса Product.
Также содержит два атрибута класса:
	•	category_count (int) — общее количество созданных категорий
	•	product_count (int) — общее количество всех товаров во всех категориях

Пример использования:

from src.category import Category
from src.product import Product

iphone = Product("iPhone 15", "Смартфон Apple", 120000.0, 15)
samsung = Product("Galaxy S24", "Смартфон Samsung", 95000.0, 10)
smartphones = Category("Смартфоны", "Мобильные устройства", [iphone, samsung])

### 🧪 Тестирование

Проект покрыт тестами, которые проверяют:
	•	корректность инициализации классов Product и Category,
	•	корректный подсчёт количества категорий и товаров,
	•	соответствие типов данных заданным требованиям.

Тесты реализованы с использованием pytest и могут быть запущены командой:
poetry run pytest -v

### 🧪 Тестирование и покрытие кода

В проекте используются pytest и pytest-cov для автоматического тестирования и генерации отчёта о покрытии кода.

Запуск тестов

Для запуска всех тестов выполните команду:
poetry run pytest -v

Запуск тестов с отчётом о покрытии

Чтобы проверить покрытие кода тестами:
poetry run pytest --cov=src --cov-report=term-missing --cov-report=html
	•	--cov=src — считает процент покрытия только в исходных файлах.
	•	--cov-report=term-missing — показывает, какие строки кода не покрыты тестами.
	•	--cov-report=html — создаёт HTML-отчёт в папке htmlcov/.
Чтобы просмотреть отчёт, открой файл htmlcov/index.html в браузере.

Сохранение текстового отчёта

Для сохранения краткого текстового отчёта в файл:
poetry run pytest --cov=src --cov-report=term > coverage_report.txt
После выполнения этой команды в корне проекта появится файл coverage_report.txt, который содержит информацию о покрытии тестами.
Этот файл добавлен в репозиторий для проверки наставником.

### 🧩 Реализованная функциональность (Lesson 2 — Классы)

В проекте реализованы классы:
- **Product** — хранит информацию о товаре (название, описание, цена, количество).
- **Category** — управляет товарами и считает общее количество категорий и продуктов.

Добавлены:
- Приватные атрибуты (`__price`, `__products`)
- Геттеры/сеттеры с валидацией цены.
- Класс-метод `new_product()`.
- Методы `add_product()` и `products` с декоратором `@property`.
- Тесты с использованием **pytest**, **Mock**, **fixtures** и **assert**.
