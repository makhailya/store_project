# Store Project

## 📦 Описание проекта
Этот проект — базовая версия системы для управления товарами и категориями.  
Он реализует классы `Product` и `Category`, которые позволяют описывать ассортимент магазина и учитывать количество товаров и категорий.

---

## ⚙️ Реализованный функционал

### 🧱 Класс `Product`
Класс описывает отдельный товар и содержит следующие атрибуты:
- **name (str)** — название товара  
- **description (str)** — описание  
- **price (float)** — цена с возможностью указания копеек  
- **quantity (int)** — количество товара на складе  

Пример использования:
```python
from src.product import Product

iphone = Product("iPhone 15", "Смартфон Apple", 120000.0, 15)

### 🗂️ Класс Category

Класс описывает категорию товаров и хранит список объектов класса Product.
Также содержит два атрибута класса:
	•	category_count (int) — общее количество созданных категорий
	•	product_count (int) — общее количество всех товаров во всех категориях

Пример использования:

from src.category import Category
from src.product import Product

iphone = Product("iPhone 15", "Смартфон Apple", 120000.0, 15)
samsung = Product("Galaxy S24", "Смартфон Samsung", 95000.0, 10)
smartphones = Category("Смартфоны", "Мобильные устройства", [iphone, samsung])

### ✨ Реализованная функциональность (Lesson 3 — Магические методы)
- Для класса `Product` реализован метод `__str__`, возвращающий строку:  
  `"Название продукта, X руб. Остаток: X шт."`
- Для класса `Category` реализован метод `__str__`, возвращающий строку:  
  `"Название категории, количество продуктов: X шт."`
- Реализован магический метод `__add__` для класса `Product`, который возвращает сумму стоимости всех товаров на складе.
- Добавлены тесты, проверяющие работу новых магических методов и совместимость с предыдущими тестами.
- Покрытие тестами сохраняется выше 75%.

Пример использования:
```python
product_a = Product("iPhone", "Смартфон Apple", 120000, 5)
product_b = Product("Galaxy", "Смартфон Samsung", 90000, 3)
print(product_a + product_b)  # 750000
print(product_a)  # iPhone, 120000 руб. Остаток: 5 шт.
```

### 📈 Lesson 6 — Улучшение архитектуры, валидация и расчётные методы
В рамках доработки проекта реализованы ключевые механизмы валидации данных и расчётной логики.

## 1. 🛡️ Валидация при инициализации Product

При создании экземпляра товара выполняется обязательная проверка количества:
Если quantity <= 0 — выбрасывается исключение ValueError с точным сообщением:
"Товар с нулевым количеством не может быть добавлен"
Проверка встроена в метод __init__ класса Product.

## 2. 📊 Метод расчёта средней цены в Category

Добавлен метод middle_price() для класса Category:
Суммирует цены всех товаров в категории и делит на количество.
Обработка крайних случаев: Если в категории нет товаров, метод возвращает 0.0.
